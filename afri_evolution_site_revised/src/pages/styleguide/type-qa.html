Accessibility (WCAG AA Compliance)

✓ Minimum Contrast Ratios

  • Headings: 4.5:1 minimum (AAA: 7:1 recommended)
  • Body Text: 4.5:1 minimum
  • Small Text (<14px): 4.5:1 minimum

✓ Responsive Typography

  • H1, H2, H3 use clamp() for fluid scaling
  • Body text: 16px base, 15px on mobile
  • Touch targets: 44×44px minimum for interactive elements

✓ Semantic HTML

  • Use <h1> through <h6> tags for proper document structure
  • Apply .h1 through .h6 classes for styling
  • Maintain logical heading hierarchy (no skipping levels)

✓ Font Features

  • Ligatures (liga) and contextual alternates (calt) enabled by default
  • Tabular numbers (tnum) opt-in via .tnums class
  • Variable fonts for optimal loading and flexibility

QA Checklist

Visual Tests

  • ☐ Verify all headings render in Manrope font
  • ☐ Verify body text renders in Source Sans 3 font
  • ☐ Test light mode contrast (all text readable)
  • ☐ Test dark mode contrast (all text readable)
  • ☐ Resize browser from 320px to 1920px (responsive scaling works)
  • ☐ Check mobile (text sizes are appropriate)

Technical Tests

  • ☐ Run WebAIM Contrast Checker on key pages
  • ☐ Validate with Chrome DevTools Lighthouse (Accessibility score)
  • ☐ Check font loading (no FOUT/FOIT)
  • ☐ Verify tabular numbers align in tables
  • ☐ Test with screen reader (NVDA/JAWS/VoiceOver)

Browser Tests

  • ☐ Chrome (latest)
  • ☐ Firefox (latest)
  • ☐ Safari (latest)
  • ☐ Edge (latest)
  • ☐ Mobile Safari (iOS)
  • ☐ Chrome Mobile (Android)
